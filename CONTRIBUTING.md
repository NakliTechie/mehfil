# Contributing to Mehfil

Thanks for your interest. Contributions are welcome — here's how things work.

## Bug reports

Open an issue on GitHub. Include:
- What you were doing
- What you expected to happen
- What actually happened
- Browser + version

## Pull requests

- Keep PRs focused — one thing per PR
- Match the existing code style (vanilla JS, no framework, no build step)
- If you're changing protocol-level behaviour, read `PROTOCOL.md` first and update it if needed
- Test across at least two browser windows before opening the PR

## Security issues

Please **do not** open a public issue for security vulnerabilities. See [SECURITY.md](SECURITY.md) for how to reach me privately.
